### 6. Minor Comments (Non-blocking)

1. **`@tailwindcss/typography@^0.5` with Tailwind v4:** The plan pins the typography plugin at `^0.5` alongside `tailwindcss@^4`. Tailwind v4 has significant API changes; the typography plugin's v4 integration may differ from v3. The plan's mitigation ("mirror shannon's provider/styles wiring") is correct, but the builder should verify the actual shannon versions during scaffold rather than trusting the pins listed here if there's a build failure. This is a "first 10 minutes of Phase 1" discovery, not a plan defect.

2. **`zustand` included but unused:** Listed in dependencies but no plan phase specifies zustand-managed state — all server data flows through TanStack Query, filters live in URL search params, and UI state is component-local. Not harmful (it's in the spec's mandated stack), but the builder shouldn't feel compelled to use it if nothing needs it.

3. **`railway.json` / `Procfile` — pick one:** Phase 6 lists both. The builder should choose based on what Railway's Nixpacks expects for a Node static site. Minor — either works.

4. **`vite preview` production caveat:** The spec and plan correctly note this is not a hardened production server (spec §5.4 documents `serve -s dist` as the drop-in alternative). This is a spec decision, not re-litigable, but worth the builder being aware that Vite's docs explicitly warn against `vite preview` in production. If Railway or traffic ever becomes a concern, the swap is trivial.
